# 1. Executive Summary

## 1.1 서비스 한 줄 정의

**RightSave Notes는 웹페이지에서 필요한 텍스트, 링크, 이미지 주소를 우클릭 한 번으로 저장하고, 크롬 확장 팝업에서 다시 확인할 수 있는 간단한 웹 자료 수집 도구이다.**

## 1.2 해결하려는 문제

학생이 과제나 발표 자료를 조사할 때 웹에서 유용한 문장, 링크, 이미지 출처를 발견해도 보통 다음과 같은 불편함이 있다.

1. 텍스트를 복사한다.
2. 메모장이나 문서를 연다.
3. 붙여넣는다.
4. 출처 URL도 따로 복사한다.
5. 나중에 다시 찾아보려면 정리가 어렵다.

이 과정은 반복될수록 시간이 오래 걸리고, 출처를 잃어버리기 쉽다.

## 1.3 핵심 해결 방법

사용자가 웹페이지에서 우클릭만 하면 현재 선택한 텍스트, 링크, 이미지 주소, 페이지 URL을 자동으로 저장한다. 저장된 항목은 확장 프로그램 팝업 UI에서 카드 형태로 확인, 삭제, 전체 삭제할 수 있다.

## 1.4 핵심 가치

- 빠른 저장: 복사/붙여넣기 없이 우클릭으로 저장
- 출처 보존: 저장 시 원본 페이지 URL 함께 저장
- 간단한 관리: 팝업 UI에서 저장 목록 확인 및 삭제
- 낮은 구현 난이도: Chrome 확장의 기본 기능을 활용하여 짧은 기간 안에 완성 가능

# 2. 서비스 정의

## 2.1 무엇을 만드는가?

Chrome 브라우저에서 작동하는 확장 프로그램을 만든다. 이 확장 프로그램은 웹페이지의 정보를 빠르게 저장하고 다시 볼 수 있게 한다.

## 2.2 누구를 위해 만드는가?

### 주요 사용자

| 사용자 유형 | 설명                                                         |
| ----------- | ------------------------------------------------------------ |
| 학생        | 과제, 발표, 보고서 작성을 위해 웹 자료를 수집하는 사용자     |
| 자료 조사자 | 여러 웹페이지의 문장, 링크, 이미지 출처를 모으는 사용자      |
| 일반 사용자 | 나중에 다시 보고 싶은 문장이나 링크를 임시 저장하려는 사용자 |

### 대표 페르소나

**이름:** 민지  
**상황:** 사회 과제 발표 자료를 준비 중이다.  
**문제:** 여러 웹사이트에서 자료를 찾지만, 좋은 문장을 발견할 때마다 문서 파일을 열고 복사해 붙여넣는 과정이 번거롭다.  
**목표:** 웹에서 찾은 자료를 빠르게 저장하고, 나중에 한 번에 확인하고 싶다.

## 2.3 서비스 목표

- 사용자가 웹페이지에서 정보를 3초 이내에 저장할 수 있게 한다.
- 저장된 정보는 원본 URL과 함께 확인할 수 있게 한다.
- 확장 프로그램의 필수 기능인 데이터 저장, 우클릭 메뉴, 팝업 UI를 모두 포함한다.
- 2026년 5월 21일 6교시까지 개발 가능한 범위로 제한한다.

# 3. 문제 정의와 기회

## 3.1 현재 문제

| 문제                        | 사용자 불편                                               |
| --------------------------- | --------------------------------------------------------- |
| 자료 복사 과정이 많음       | 텍스트 복사, 메모장 열기, 붙여넣기 반복                   |
| 출처 누락 가능성            | 내용만 저장하고 URL을 빼먹기 쉬움                         |
| 저장 자료가 흩어짐          | 메모장, 카카오톡, 문서 파일 등에 분산됨                   |
| 이미지 출처 저장이 번거로움 | 이미지 자체보다 이미지 주소나 페이지 URL이 필요할 때 불편 |

## 3.2 기회

Chrome 확장 프로그램은 브라우저 사용 흐름 안에서 작동하기 때문에 사용자가 별도 앱을 열 필요가 없다. 특히 우클릭 메뉴는 사용자가 웹페이지를 탐색하는 흐름을 끊지 않고 기능을 제공할 수 있다.

## 5.3 상세 기능 명세

## F-01. 우클릭 메뉴 생성

### 목적

사용자가 웹페이지에서 저장하고 싶은 대상을 발견했을 때 즉시 저장할 수 있게 한다.

### 메뉴명

`RightSave에 저장하기`

### 메뉴 표시 위치

| 상황                   | context     |
| ---------------------- | ----------- |
| 텍스트 선택 후 우클릭  | `selection` |
| 링크 위에서 우클릭     | `link`      |
| 이미지 위에서 우클릭   | `image`     |
| 일반 페이지에서 우클릭 | `page`      |

### 완료 조건

- 확장 프로그램 설치 또는 새로고침 후 우클릭 메뉴가 나타난다.
- 메뉴 클릭 시 `background.js`의 클릭 이벤트가 실행된다.

---

## F-02. 데이터 저장

### 목적

사용자가 저장한 웹 정보를 브라우저 확장 저장소에 보관한다.

### 저장 위치

`chrome.storage.local`

### 저장 키

`savedItems`

### 데이터 구조

```json
{
  "id": "item_1716000000000",
  "type": "selection",
  "content": "저장한 내용",
  "pageUrl": "https://example.com/article",
  "pageTitle": "예시 페이지 제목",
  "createdAt": "2026-05-18T13:00:00.000Z"
}
```

### 필드 설명

| 필드        | 타입   | 설명                                         |
| ----------- | ------ | -------------------------------------------- |
| `id`        | string | 항목 고유 ID                                 |
| `type`      | string | `selection`, `link`, `image`, `page` 중 하나 |
| `content`   | string | 저장된 실제 내용                             |
| `pageUrl`   | string | 저장 당시 페이지 주소                        |
| `pageTitle` | string | 저장 당시 탭 제목                            |
| `createdAt` | string | ISO 형식 저장 시각                           |

### 완료 조건

- 저장된 데이터는 브라우저를 닫았다 열어도 유지된다.
- 새 항목은 기존 배열 맨 앞에 추가된다.
- 저장할 내용이 비어 있으면 저장하지 않는다.

---

## F-03. 팝업 UI

### 목적

저장된 항목을 사용자가 쉽게 확인하고 관리하게 한다.

### 화면 구성

| 영역        | 내용                                |
| ----------- | ----------------------------------- |
| Header      | 앱 이름, 전체 삭제 버튼             |
| Summary     | 저장된 항목 개수 표시               |
| Filter      | 전체/텍스트/링크/이미지/페이지 필터 |
| List        | 저장 항목 카드 목록                 |
| Empty State | 저장된 항목이 없을 때 안내 문구     |

### 카드 구성

- 타입 배지
- 저장 내용
- 원본 페이지 제목
- 원본 URL
- 저장 시간
- 삭제 버튼

### 완료 조건

- 저장된 항목이 최신순으로 표시된다.
- 긴 텍스트와 긴 URL은 줄바꿈되어 UI가 깨지지 않는다.
- 항목이 없으면 “저장된 항목이 없습니다.” 안내가 표시된다.

---

## F-04. 삭제 기능

### 개별 삭제

- 각 카드의 `삭제` 버튼 클릭
- 해당 인덱스 또는 ID의 항목 삭제
- 저장소 업데이트 후 화면 다시 렌더링

### 전체 삭제

- 상단 `전체 삭제` 버튼 클릭
- 확인창 표시
- 사용자가 확인하면 `savedItems`를 빈 배열로 저장

# 6. UX 설계

## 6.1 핵심 UX 원칙

1. **흐름을 끊지 않는다.**  
   사용자는 현재 보고 있는 웹페이지에서 바로 저장할 수 있어야 한다.

2. **저장 결과를 쉽게 확인한다.**  
   팝업을 열면 저장 목록이 바로 보여야 한다.

3. **복잡한 설정을 요구하지 않는다.**  
   로그인, 서버, 동기화 없이 설치 후 바로 사용할 수 있어야 한다.

4. **출처를 잃어버리지 않는다.**  
   저장 내용과 함께 원본 URL을 자동 저장한다.

---

## 6.2 사용자 여정

| 확인 | 확장 아이콘 클릭              | 팝업 목록 표시 | 저장 자료 확인 |
| 관리 | 삭제 버튼 클릭                | 항목 삭제      | 목록 정리      |

## 8.2 background.js

```javascript
const MENU_ID = "save-to-rightsave";

chrome.runtime.onInstalled.addListener(() => {
  chrome.contextMenus.create({
    id: MENU_ID,
    title: "RightSave에 저장하기",
    contexts: ["selection", "link", "image", "page"],
  });
});

chrome.contextMenus.onClicked.addListener(async (info, tab) => {
  if (info.menuItemId !== MENU_ID) return;

  const item = createSaveItem(info, tab);

  if (!item.content) {
    console.warn("저장할 내용이 없습니다.");
    return;
  }

  try {
    const result = await chrome.storage.local.get({ savedItems: [] });
    const savedItems = Array.isArray(result.savedItems)
      ? result.savedItems
      : [];

    savedItems.unshift(item);

    await chrome.storage.local.set({ savedItems });
  } catch (error) {
    console.error("RightSave 저장 실패:", error);
  }
});

function createSaveItem(info, tab) {
  const pageUrl = tab?.url || "";
  const pageTitle = tab?.title || "";

  let type = "page";
  let content = pageUrl;

  if (info.selectionText) {
    type = "selection";
    content = info.selectionText.trim();
  } else if (info.linkUrl) {
    type = "link";
    content = info.linkUrl;
  } else if (info.srcUrl) {
    type = "image";
    content = info.srcUrl;
  }

  return {
    id: `item_${Date.now()}`,
    type,
    content,
    pageUrl,
    pageTitle,
    createdAt: new Date().toISOString(),
  };
}
```

## 8.4 popup.js

```javascript
const itemList = document.getElementById("itemList");
const summary = document.getElementById("summary");
const clearAllBtn = document.getElementById("clearAllBtn");
const filterButtons = document.querySelectorAll(".filter");

let savedItems = [];
let currentFilter = "all";

async function loadItems() {
  const result = await chrome.storage.local.get({ savedItems: [] });
  savedItems = Array.isArray(result.savedItems) ? result.savedItems : [];
  renderItems();
}

function renderItems() {
  const filteredItems =
    currentFilter === "all"
      ? savedItems
      : savedItems.filter((item) => item.type === currentFilter);

  summary.textContent = `저장된 항목 ${savedItems.length}개`;
  itemList.innerHTML = "";

  if (filteredItems.length === 0) {
    itemList.innerHTML = `<p class="empty">저장된 항목이 없습니다.</p>`;
    return;
  }

  filteredItems.forEach((item) => {
    const card = document.createElement("section");
    card.className = "item-card";

    card.innerHTML = `
      <div class="card-top">
        <span class="badge">${getTypeLabel(item.type)}</span>
        <button class="delete-btn" data-id="${item.id}">삭제</button>
      </div>
      <p class="content">${escapeHtml(item.content)}</p>
      <p class="source">${escapeHtml(item.pageTitle || "제목 없음")}</p>
      <p class="url">${escapeHtml(item.pageUrl)}</p>
      <time>${formatDate(item.createdAt)}</time>
    `;

    itemList.appendChild(card);
  });

  document.querySelectorAll(".delete-btn").forEach((button) => {
    button.addEventListener("click", async () => {
      const id = button.dataset.id;
      savedItems = savedItems.filter((item) => item.id !== id);
      await chrome.storage.local.set({ savedItems });
      renderItems();
    });
  });
}

function getTypeLabel(type) {
  const labels = {
    selection: "텍스트",
    link: "링크",
    image: "이미지",
    page: "페이지",
  };

  return labels[type] || "기타";
}

function formatDate(isoString) {
  const date = new Date(isoString);
  return date.toLocaleString("ko-KR");
}

function escapeHtml(text) {
  return String(text)
    .replaceAll("&", "&amp;")
    .replaceAll("<", "&lt;")
    .replaceAll(">", "&gt;")
    .replaceAll('"', "&quot;")
    .replaceAll("'", "&#039;");
}

filterButtons.forEach((button) => {
  button.addEventListener("click", () => {
    filterButtons.forEach((btn) => btn.classList.remove("active"));
    button.classList.add("active");

    currentFilter = button.dataset.type;
    renderItems();
  });
});

clearAllBtn.addEventListener("click", async () => {
  const ok = confirm("저장된 항목을 모두 삭제할까요?");
  if (!ok) return;

  savedItems = [];
  await chrome.storage.local.set({ savedItems });
  renderItems();
});

loadItems();
```

# 11. 성공 기준

## 11.1 기능 성공 기준

- 확장 프로그램이 Chrome 개발자 모드에서 정상 로드된다.
- 우클릭 메뉴가 정상 생성된다.

- 개별 삭제와 전체 삭제가 작동한다.

## 11.2 사용자 경험 성공 기준

- 사용자는 3번 이하의 행동으로 정보를 저장할 수 있다.
  1. 저장 대상 선택 또는 우클릭
  2. 메뉴 클릭
  3. 팝업에서 확인
- 저장한 내용과 출처를 함께 확인할 수 있다.
- 저장된 항목이 없을 때도 사용자가 상황을 이해할 수 있다.

# 13. 발표용 요약

## 프로젝트 소개

RightSave Notes는 웹페이지에서 중요한 텍스트, 링크, 이미지 주소를 우클릭으로 저장하고, 확장 프로그램 팝업에서 다시 확인할 수 있는 Chrome 확장 프로그램이다.

## 만든 이유

과제나 발표 자료를 조사할 때 웹에서 찾은 내용을 따로 복사하고 출처를 저장하는 과정이 번거롭기 때문에, 브라우저 안에서 바로 저장할 수 있는 도구가 필요하다고 생각했다.

## 핵심 기능

1. 우클릭 메뉴 생성
2. 선택 텍스트, 링크, 이미지, 페이지 URL 저장
3. 팝업 UI에서 저장 목록 확인
4. 개별 삭제 및 전체 삭제

## 사용 기술

- Chrome Extension Manifest V3
- `chrome.contextMenus`
- `chrome.storage.local`
- HTML, CSS, JavaScript

## 기대 효과

자료 수집 시간을 줄이고, 출처가 포함된 웹 메모를 쉽게 관리할 수 있다.
